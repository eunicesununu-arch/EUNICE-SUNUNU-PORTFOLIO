# Deployment Guide

## How to Deploy Your Website

### Option 1: Traditional Web Hosting
1. **Choose a hosting provider** (examples: Bluehost, HostGator, GoDaddy)
2. **Purchase a domain name** (e.g., eunicesununu.com)
3. **Upload files via FTP**:
   - Use an FTP client like FileZilla
   - Connect to your hosting server
   - Upload all website files to the public_html folder
4. **Test your website** by visiting your domain

### Option 2: GitHub Pages (Free)
1. **Create a GitHub account** if you don't have one
2. **Create a new repository** named `username.github.io`
3. **Upload all website files** to the repository
4. **Enable GitHub Pages** in repository settings
5. **Access your site** at `https://username.github.io`

### Option 3: Netlify (Free)
1. **Sign up at netlify.com**
2. **Drag and drop** your entire website folder
3. **Netlify will automatically deploy** your site
4. **Get a custom URL** (e.g., your-site-name.netlify.app)

### Option 4: Vercel (Free)
1. **Sign up at vercel.com**
2. **Import your project** from GitHub or upload files
3. **Vercel will deploy** automatically
4. **Get a professional URL**

## Troubleshooting Common Issues

### If images don't load:
- Check file paths in HTML
- Verify image file names match exactly
- Ensure images are in the correct folder

### If contact form doesn't work:
- Verify WhatsApp number format
- Check JavaScript console for errors
- Test form on different browsers

### If layout looks broken:
- Check CSS file links
- Verify Bootstrap CDN is accessible
- Test on different screen sizes
